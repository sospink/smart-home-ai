## 修复的问题

### 1. ✅ Nginx 反向代理配置错误（关键）
**问题：** Nginx 的 rewrite 规则将 `/api/v1` 重写为 `/v1`，导致后端路由404

**修复：**
```nginx
# 错误 ❌
location /api {
    rewrite ^/api/(.*) /$1 break;  # 这会去掉 /api 前缀
    proxy_pass http://backend;
}

# 正确 ✅
location /api {
    proxy_pass http://backend;     # 保持 /api 前缀
}
```

### 2. ✅ 数据库驱动协议错误
**问题：** docker-compose.yml 使用 `mysql+pymysql://`，但后端使用异步 SQLAlchemy 需要 `mysql+aiomysql://`

**修复：**
```yaml
# 错误 ❌
DATABASE_URL: mysql+pymysql://...

# 正确 ✅
DATABASE_URL: mysql+aiomysql://...
```

### 3. ✅ admin 密码初始化
**问题：** 数据库中 admin 密码哈希与 "123456" 不匹配

**修复：** 执行 SQL 脚本重置密码

### 4. ✅ 前端环境变量配置错误（关键）
**问题：** 
- 变量名不匹配：前端代码使用 `NEXT_PUBLIC_API_BASE_URL`，docker-compose 配置为 `NEXT_PUBLIC_API_URL`
- Next.js 的 `NEXT_PUBLIC_` 变量必须在**构建时**注入，不是运行时

**修复：**
```yaml
# docker-compose.yml
frontend:
  build:
    context: ./smart-home-ai-frontend
    dockerfile: Dockerfile
    args:                           # ✅ 新增：构建参数
      NEXT_PUBLIC_API_BASE_URL: ${NEXT_PUBLIC_API_BASE_URL}
      NEXT_PUBLIC_WS_URL: ${NEXT_PUBLIC_WS_URL}
```

```dockerfile
# Dockerfile (frontend)
FROM node:22-alpine AS builder
# 构建时注入环境变量
ARG NEXT_PUBLIC_API_BASE_URL      # ✅ 新增
ARG NEXT_PUBLIC_WS_URL            # ✅ 新增
ENV NEXT_PUBLIC_API_BASE_URL=${NEXT_PUBLIC_API_BASE_URL}
ENV NEXT_PUBLIC_WS_URL=${NEXT_PUBLIC_WS_URL}

# 删除本地配置文件，防止覆盖 ARG 传入的值
RUN rm -f .env.local .env         # ✅ 新增
```

```bash
# .env
NEXT_PUBLIC_API_BASE_URL=http://172.19.4.213:8088/api   # ✅ 修正变量名
NEXT_PUBLIC_WS_URL=ws://172.19.4.213:8088/ws
```

```bash
# .env.local（同步修改为生产配置）
NEXT_PUBLIC_API_BASE_URL=http://172.19.4.213:8088/api   # ✅ 原为 localhost:8000
NEXT_PUBLIC_WS_URL=ws://172.19.4.213:8088/ws
```

### 6. ✅ .next 构建缓存问题（根因！）
**问题：** `.next` 目录包含之前构建的缓存文件，里面硬编码了 `localhost:8000`，Docker 构建时复用了这些缓存。

**修复：**
1. Dockerfile 新增 `RUN rm -rf .next node_modules` 清理缓存
2. `next.config.ts` 新增 `env` 配置显式声明环境变量

```dockerfile
# Dockerfile
RUN rm -rf .next node_modules  # ✅ 清理旧缓存
```

```typescript
// next.config.ts
const nextConfig: NextConfig = {
  output: 'standalone',
  env: {
    NEXT_PUBLIC_API_BASE_URL: process.env.NEXT_PUBLIC_API_BASE_URL || '/api',
    NEXT_PUBLIC_WS_URL: process.env.NEXT_PUBLIC_WS_URL || '',
  }
};
```
