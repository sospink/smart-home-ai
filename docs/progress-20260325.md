## 本次迭代完成内容

### 1. AI 对话模块 — 全链路联调完成 ✅

- **Dify SSE 流式对话**: 支持 `thinking` + `message` + `message_end` 事件解析
- **会话管理**: 创建/列表/删除会话，支持会话切换
- **消息历史**: 查看历史消息、Markdown 渲染、代码高亮
- **多模型支持**: 管理端一键切换 Ollama 模型
- **打字指示器**: 实时 typing 动画、流式 token 渲染

### 2. 设备控制模块 — HA 完整对接 ✅

- **11 个逻辑设备**: 客厅灯/卧室灯/书房灯/空调/咖啡机/热水器/前门锁/窗帘/温度/湿度/PM2.5
- **HA 虚拟设备**: 18 个 input_boolean/select/number/sensor 实体 + HA 配置 YAML
- **设备接口**: 列表(GET) + 详情(GET) + 控制(POST) + 服务列表(GET) + 场景激活(POST)
- **操作日志**: 每次控制自动写入 `operation_logs` 表，记录用户/设备/操作
- **前端 UI**: 统计卡片 + 分类筛选 + 设备卡片网格 + Toggle 控制 + 场景预设

### 3. 监控大屏 — 三阶段全部完成 ✅

#### Phase 1: 核心数据接口
| 接口 | 数据源 | 说明 |
|------|--------|------|
| `/monitor/resources` | psutil | CPU/内存/GPU/磁盘实时监控 |
| `/monitor/ai-status` | Ollama API | 模型名/状态/参数规模/量化方式 |
| `/monitor/services` | HTTP 健康检测 | 5 个服务（FastAPI/Ollama/HA/Dify/InfluxDB） |
| `/monitor/dashboard` | 多源聚合 | 一次请求返回全部监控数据 |

#### Phase 2: 统计数据接口
| 接口 | 数据源 | 说明 |
|------|--------|------|
| `/monitor/activity` | Dify + MySQL + Ollama | 今日对话/控制/知识库/模型统计 |
| `/monitor/control-trends` | MySQL 聚合 | 操作趋势（今日 vs 昨日按小时） |
| `/monitor/logs` | operation_logs 表 | 真实操作日志（登录+设备控制） |

#### Phase 3: 历史数据接口
| 接口 | 数据源 | 说明 |
|------|--------|------|
| `/monitor/sensor-history` | HA History API + 模拟混合 | 温度/湿度/PM2.5 折线图（96个点） |
| 登录日志埋点 | auth.py → operation_log | 每次登录自动记录 |
| HA History 方法 | homeassistant.py | `get_history()` 查询 HA 内置历史 |

#### 前端渲染
- 纯 SVG 手绘图表（零依赖）
- 环形进度图（CPU/内存/GPU/磁盘）
- 柱状图（操作趋势 今日/昨日）
- 面积折线图（传感器历史 温度/湿度/PM2.5）
- 所有数据全部从后端 API 获取，**零 Mock**

### 4. 管理后台模块 ✅

- **知识库管理**: Dify Knowledge API 完整代理（数据集列表/创建/删除/文档管理）
- **模型管理**: Ollama API 封装（列表/详情/下载/删除/运行状态）
- **用户管理**: CRUD + 角色管理 + 启用/禁用
- **系统设置**: 外部服务连接测试 + 配置管理
- **仪表盘**: 系统概览 + 服务状态

### 5. 基础设施改进

| 改进 | 说明 |
|------|------|
| `config.yaml` 统一配置 | 替代分散的 .env，支持 YAML → env → 默认值 三级优先级 |
| 操作日志系统 | `operation_log` 模型 + 自动埋点（登录/设备控制） |
| 超时保护 | 外部 API（Dify/HA/Ollama）加 `asyncio.wait_for` 超时 |
