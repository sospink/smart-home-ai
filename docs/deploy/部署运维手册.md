## 第一章 环境要求与快速部署

### 1.1 硬件要求

| 配置项 | 最低要求 | 推荐配置 |
|---|---|---|
| CPU | ≥16 核 24 线程 | Intel i9-13980HX / AMD Ryzen 9 7945HX |
| 内存 | ≥32G DDR5 | 64G |
| 存储 | ≥512G NVMe SSD | 2TB |
| GPU | ≥16G 显存，≥4608 计算核心 | NVIDIA RTX 4080/4090 |
| 操作系统 | Ubuntu 22.04 LTS | Ubuntu 22.04 LTS Server |
| CUDA | ≥12.0 | CUDA 12.4 + cuDNN 8.9+ |

### 1.2 软件依赖

```bash
# Docker & Compose
sudo apt-get update && sudo apt-get install -y ca-certificates curl gnupg
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
sudo usermod -aG docker $USER && newgrp docker
docker --version && docker compose version

# NVIDIA Container Toolkit（GPU 支持）
distribution=$(. /etc/os-release;echo $ID$VERSION_ID)
curl -s -L https://nvidia.github.io/nvidia-docker/gpgkey | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg
curl -s -L https://nvidia.github.io/nvidia-docker/$distribution/nvidia-docker.list | sudo tee /etc/apt/sources.list.d/nvidia-docker.list
sudo apt-get update && sudo apt-get install -y nvidia-container-toolkit
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker
docker run --rm --gpus all nvidia/cuda:12.4-base-ubuntu22.04 nvidia-smi
```

### 1.3 获取项目与配置

```bash
git clone <项目仓库URL> smart-home-llm && cd smart-home-llm
```

## 第二章 模型管理

### 2.1 模型推荐

| 模型 | 量化版本 | 显存占用 | 用途 |
|---|---|---|---|
| Qwen2.5-7B-Instruct | Q4_K_M | ~5.0G | 主力对话 |
| DeepSeek-R1-7B | Q4_K_M | ~4.7G | 推理任务 |
| BAAI/bge-m3 | FP16 | ~1.1G | 文本 Embedding |
| Qwen2.5-14B | Q4_K_M | ~9.5G | 需 ≥16G 显存 |

### 2.2 常用命令

```bash
docker exec ollama ollama pull qwen2.5:7b-instruct-q4_K_M  # 拉取模型
docker exec ollama ollama pull bge-m3                        # 拉取 Embedding
docker exec ollama ollama list                               # 已安装列表
docker exec ollama ollama ps                                 # 运行状态
docker exec ollama ollama rm <model_name>                    # 删除模型
```

### 2.3 Dify 中配置模型

1. 登录 Dify Web UI → 设置 → 模型供应商 → 选择 Ollama
2. Base URL 填写 `http://ollama:11434`（Docker 内网地址，不用 localhost）
3. 添加模型名称（如 `qwen2.5:7b-instruct-q4_K_M`）
4. Embedding 模型：类型选 Text Embedding，名称填 `bge-m3`

## 第三章 Home Assistant 与电子沙盘

### 3.1 HA 初始化

1. 访问 `http://服务器IP:8123`，完成设置向导
2. 创建管理员账号，设置时区
3. 头像 → 安全 → 长期访问令牌 → 创建
4. 将 Token 配置到 `config.yaml` 的 `services.homeassistant.token`

### 3.2 虚拟设备配置

HA 中通过 `configuration.yaml` 的 `template:` 配置虚拟设备：
- 灯光（`light.*`）：使用 `input_boolean` + `input_number`(brightness) 映射
- 开关（`switch.*`）：使用 `input_boolean` 映射
- 传感器（`sensor.*`）：使用 `input_number` 映射
- 场景（`scene.*`）：HA 原生场景

### 3.3 设备控制 API

```bash
# 获取所有设备状态
curl "http://localhost:8000/api/v1/devices/" -H "Authorization: Bearer <JWT>"

# 控制设备
curl -X POST "http://localhost:8000/api/v1/devices/input_boolean.light_living_room/control" \
  -H "Authorization: Bearer <JWT>" \
  -H "Content-Type: application/json" \
  -d '{"service": "toggle"}'

# 沙盘数据注入
curl -X POST "http://localhost:8000/api/v1/sandbox/inject" \
  -H "Content-Type: application/json" \
  -d '{"entity_id": "sensor.shi_nei_wen_du", "value": "28.5", "source": "manual"}'
```

## 第四章 日常运维

### 4.1 启停管理

```bash
docker compose up -d                     # 启动
docker compose down                      # 停止
docker compose restart <service>         # 重启单个服务
docker compose ps                        # 状态
docker compose logs --tail=200 <service> # 日志
docker compose logs -f dify-api          # 实时日志
```

### 4.2 数据备份

```bash
BACKUP_DIR="./backups/$(date +%Y%m%d_%H%M%S)"
mkdir -p $BACKUP_DIR
docker exec db pg_dump -U postgres dify > $BACKUP_DIR/dify_db.sql
cp -r ./ha-config $BACKUP_DIR/ha_config
echo "备份完成：$BACKUP_DIR"
```

### 4.3 升级流程

```bash
bash scripts/backup.sh                   # 1. 备份
docker compose pull                      # 2. 拉取新镜像
docker compose up -d --force-recreate    # 3. 重建启动
docker compose ps && docker compose logs --tail=50  # 4. 验证
```

### 4.4 资源监控

```bash
docker stats           # 容器资源占用
nvidia-smi             # GPU 使用
df -h && du -sh volumes/  # 磁盘空间
docker system prune -f    # 清理（慎用）
```

## 第五章 常见故障排查

| 现象 | 可能原因 | 排查方法 | 解决方案 |
|---|---|---|---|
| Ollama 无响应 | 服务未启动/GPU 问题 | `docker logs ollama` | `docker restart ollama`，检查 `nvidia-smi` |
| Dify 无法访问 | API 容器异常 | `docker ps \| grep dify` | `docker compose restart dify-api dify-web` |
| HA 无法访问 | 配置错误 | `docker logs homeassistant` | 检查 `configuration.yaml` 语法 |
| LLM 推理慢 | 未使用 GPU/量化不当 | `ollama ps` / `nvidia-smi` | 确认 CUDA 可用，切换 Q4_K_M |
| 前端 API 404 | Nginx rewrite 错误 | 检查 `nginx.conf` | `proxy_pass` 保留 `/api` 前缀 |
| 前端环境变量无效 | 构建时未注入 | `docker exec frontend env` | Dockerfile 中 `ARG` + `RUN rm -f .env.local` |
